# 副業を続けるために、まず本業をAIで30分縮めることにした話

副業で稼ぎたい。でも本業から帰ってきた頃にはもう頭が動かない。週末は寝て終わる。──そういう人にこそ、副業ノウハウより先に試してほしいのが、本業をAIで30分早く終わらせる技術です。

私はメーカーの商品開発職として働きながら、note副業を5ヶ月続けています。フォロワーはまだ200人。「成功者」とは到底呼べない規模ですが、その分、続けるためのリアルな工夫だけは溜まってきました。今日はその中で、一番効いた話を共有します。

## 第1章: なぜ副業志向の人ほど「本業のAI効率化」から始めるべきか

副業を始めて最初の2ヶ月、私はほとんど記事を書けませんでした。書こうと思って机に座っても、頭の中が本業の残務でいっぱいで、画面を10分眺めて寝室に行く日が続きます。才能の問題ではなく、単純にエネルギーが残っていなかったのだと、今は分かります。

国内の各種就業実態調査でも、副業希望者が挙げる最大の障壁として「時間が足りない」が上位に来るとされており(出典: 業界紙 Web 等)、これは私個人の問題ではなく、構造的な課題のようです。つまり、副業ノウハウを学ぶ前に、本業で消耗しきっている状態をどう変えるかが先決ということになります。

私が出した結論はシンプルでした。**副業を増やすのではなく、本業を縮めるところから始める**。1日30分、月20営業日で10時間。これだけ捻出できれば、副業の助走には十分だと思っています。

なお本記事では、副業のジャンル選びや収益化テクには踏み込みません。あくまで「本業時間をAIで圧縮する」という、その手前の話だけを書きます。

## 第2章: 削れる30分を見つける — 業務の可視化から始める

AIを入れる前にやってほしいのが、自分の業務の見える化です。「忙しい」という感覚は意外と当てになりません。まずは1週間、15分粒度で業務ログを取ってみます。手書きでもスプレッドシートでも構いません。

国内ホワイトカラーの業務時間調査によれば、文章作成や情報整理など「言語に関わる作業」が業務時間の3〜4割を占めるとされています(出典: 業界紙 Web 等)。この領域はAIと相性が良く、削減候補として最も狙いやすい場所です。

私の場合、ログを取ってみて初めて気づいたのが、メール返信に1日40分も使っていたことでした。削減候補は3つに分類すると整理しやすくなります。文章系(メール、議事録、報告書)、情報整理系(リサーチ、論点整理)、定型処理系(フォーマット変換、集計)の3つです。

最初に削るべきは、毎日必ず発生していて、自分の判断比率が低い業務。私の場合は、定型的な返信メールでした。ここから手を付けるのが結局一番早いと思います。

## 第3章: 文章系の仕事をAIに任せる — メール・議事録・報告書

文章系は一番効果が出やすく、社内ルール上も始めやすい領域です。生成AIの業務利用に関する調査でも、用途として「メールの下書き」「議事録の要約」が上位に来ると報告されています(出典: 業界紙 Web 等)。私もここから入りました。

返信メールは、相手のメール本文と「丁寧めのトーンで、結論先行、3行以内」のような指示を渡すだけで、8割の精度で下書きが出ます。残り2割を自分で直す運用にすると、1通あたり3〜5分が1分に縮みました。

議事録は、録音アプリの文字起こしをAIに渡し、「決定事項」「ToDo(担当者付き)」「持ち帰り課題」の3項目に整理してもらうのが定番です。商品開発の会議だと仕様変更の議論で論点が散りがちなのですが、この型に流し込むと、後から見返したときの読みやすさが段違いに変わります。

報告書は丸投げせず、骨子だけ自分で書いてAIに肉付けしてもらう逆方向の使い方が安全です。**最終的な数字や評価コメントは、必ず自分の判断で書き直す**ようにしています。

## 第4章: 情報整理と思考の壁打ちに使う

文章系で時間が浮いてきたら、次は思考の壁打ちです。AIは「答えを出す道具」ではなく、自分の頭を整理する相棒として使う方が、私には合いました。

経営層向けの調査でも、生成AIを「アイデア出しのパートナー」として活用する企業が増えているとされています(出典: 業界紙 Web 等)。私自身、会議の30分前に議題と前提情報をAIに渡し、「想定される論点を5つ挙げて」と頼むのを習慣にしています。

商品開発の現場では、新素材の採用可否を議論するときなど、論点が10も20も浮かんでしまうことがあります。AIに先に整理してもらうと、自分が考えていなかった切り口(コスト、調達リスク、規制動向など)が拾えて、会議での発言の質が上がった実感があります。

ただし、AIの答えを鵜呑みにすると、当たり障りのないアウトプットしか出てきません。私は必ず一度、自分の言葉で書き直す工程を挟んでいます。これを省くと、徐々に「自分の意見」が痩せていく感覚があるので、ここは譲らないようにしています。

## 第5章: 1.3倍を支えるのは時間より「集中の質」

ここで一つ、効率化の罠について書きます。30分浮かせると、その時間に余計な仕事が流れ込んでくる現象です。「手が空いてるなら」と頼まれごとが増え、結局退社時間は変わらない。業務効率化を扱った記事でもしばしば指摘されている話です(出典: 業界紙 Web 等)。

対策はシンプルで、浮いた30分は最初からブロックして予定に入れてしまうことです。私の場合、17:30〜18:00を「集中時間」とカレンダーに固定し、その間は会議も依頼も入れない運用にしました。最初は気まずいですが、1ヶ月もすれば周囲も慣れます。

「1.3倍」という言葉を見出しに入れましたが、これはざっくりとした計算です。月160時間労働の人が30分×20営業日=10時間を生み出せれば、単純計算で約1.06倍。さらに浮いた時間を判断業務に寄せると、体感的には1.2〜1.3倍くらいに感じる、という話で、厳密な数値ではありません。

大事なのは時間の量より、その時間に**何を寄せるか**だと思っています。評価される仕事(提案、企画、判断)に寄せていけば、本業の評価も自然と付いてきます。

## 第6章: 明日から試す3ステップと、私の失敗談

最後に、明日から試せる3ステップをまとめます。生成AIを継続的に業務利用している人ほど、「小さく始めた」と回答する割合が高いという調査もあります(出典: 業界紙 Web 等)。

ステップ1は、1業務だけAIに置き換えること。全部いっぺんにやろうとすると挫折します。おすすめは、毎日発生する短文メールの返信です。

ステップ2は、1週間続けて時間計測すること。感覚ではなく数字で「本当に縮んだか」を確認します。これをやらないと、効果があった気になって終わります。

ステップ3は、削れた時間を副業ブロックとして予定に固定すること。曖昧にしておくと、本業の残務に吸い込まれます。

正直な失敗談も3つだけ書いておきます。1つ目は、最初の1ヶ月、AIで時短できた分をすべて本業の追加タスクに使ってしまったこと。2つ目は、プロンプトを毎回ゼロから書いていて、結局時間が増えていた時期があったこと(テンプレ化で解決しました)。3つ目は、機密情報を含む文書をうっかり貼りそうになったこと。社内ルールの確認は最初にやっておくのが安全です。

副業は、才能でもネタでもなく、続けられる体力の確保から始まると、5ヶ月やってみて思っています。本業を30分縮める習慣が、その土台になります。
